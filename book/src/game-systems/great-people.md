# Great People & Great Works

## Great People

Great People are unique units with powerful one-time abilities.

### Definitions

```rust
struct GreatPersonDef {
    name: &'static str,
    person_type: GreatPersonType,
    era: AgeType,
    retire_effect: RetireEffect,
}
```

### Types

| Type | Generated By |
|------|-------------|
| Great General | Encampment (land military) |
| Great Admiral | Harbor (naval military) |
| Great Engineer | Industrial Zone |
| Great Merchant | Commercial Hub |
| Great Musician | Theater Square |
| Great Artist | Theater Square |
| Great Writer | Theater Square |
| Great Prophet | Holy Site |
| Great Scientist | Campus |

### Built-in Great Persons

| Name | Type | Retire Effect |
|------|------|--------------|
| Sun Tzu | General | +5 land combat strength (permanent modifier) |
| Themistocles | Admiral | +5 naval combat strength (permanent modifier) |
| Imhotep | Engineer | +200 production to nearest city |
| Marco Polo | Merchant | +200 gold to civilization |

### Retirement

`RulesEngine::retire_great_person()` applies the effect and marks the unit as retired:

- `CombatStrengthBonus { domain, amount }` -- adds a permanent modifier to the civ's `great_person_modifiers`, applied in all future combat
- `ProductionBurst { amount }` -- adds production to the nearest city's `production_stored`
- `GoldGrant { amount }` -- adds gold to the civ's treasury

A great person can only be retired once. Attempting to retire an already-retired person returns an error.

### Combat Integration

Great person combat modifiers (from retirement) are included in the combat strength calculation during `attack()`. For example, after retiring Sun Tzu, all land combat units get +5 CS.

### Not Yet Implemented

- Great person points accumulation per turn (based on districts)
- Candidate pool with era gating
- `recruit_great_person()` action
- Great person points competition between civilizations

## Great Works

Great Works are cultural artifacts that generate tourism and culture.

### Types

```rust
enum GreatWorkType {
    Writing,    // 2 tourism
    Art,        // 3 tourism
    Music,      // 4 tourism
    Relic,      // 4 tourism
    Artifact,   // 3 tourism
}
```

### Slots

Great works are placed into slots in cities:

```rust
struct GreatWorkSlot {
    slot_type: GreatWorkSlotType,  // Writing, Art, Music, Any
    work: Option<GreatWorkId>,
}
```

Slot types enforce which work types can be placed:
- `Writing` slots accept only Writing works
- `Art` slots accept only Art works
- `Music` slots accept only Music works
- `Any` slots accept all types

Buildings define their great work slots in `BuildingDef.great_work_slots`. When a building is constructed, its slots become available on the city.

### Tourism Generation

Each great work contributes its tourism value to the owning civilization's `tourism_output`, which accumulates into `tourism_accumulated` each turn. This feeds the [Cultural Victory](./victory.md) condition.
